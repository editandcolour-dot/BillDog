### What Was Built
- **Database:** `001_initial_schema.sql` (profiles, cases, municipalities) with strict RLS policies.
- **Middlewares & Config:** `middleware.ts` for route protection and session refresh. `.env.local` configured.
- **Routing:** Auth callback route (`app/auth/callback/route.ts`) to exchange verification codes for cookies. NextJS route groups mapped securely.
- **Forms & Actions:** `SignupForm`, `LoginForm`, `OnboardingForm`, and `app/actions/auth.ts`.

### Bug Investigation & Resolution
- **Issue:** New signups couldn't save onboarding data. `OnboardingForm` executed client-side updates that failed silently because `auth.uid()` evaluates differently before/after email verification cookies settle.
- **Fix:** Refactored Onboarding to a server action (`completeOnboardingAction`) using `UPSERT`. This guarantees absolute session integrity reading directly from server-side cookies, bypassing browser inconsistencies.
- **Note:** Supabase auth email confirmation was temporarily toggled **OFF** to bypass rate limits during testing. **MUST TOGGLE BACK ON FOR PROD.**
